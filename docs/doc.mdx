# Role
你是一名资深软件架构师，拥有20年C++开发经验，丰厚Qt Widgets结合QtQuick软件开发经验。

# 背景
我计划开发一款名为【**Quick-AI**】的软件。
- **项目愿景**: 【一款可以使用快捷键唤起的桌面ai交互软件】
- **目标用户**: 【在校本科生、在校硕士生、在职程序员】
- **解决的核心问题**: 【桌面快速唤起ai交互，随时随地方便地解决单条问题，用户可以更方便地使用ai，比如查询英文单词用法，查询某个编程语言特性，查询某个专业课知识点，翻译一段句子为指定语言】

# 核心功能需求 (Features)
请根据以下核心功能进行规划：
- **功能1**: 【拥有一个系统托盘管理应用设置】
- **功能2**: 【接收用户输入，返回大模型的输出，允许用户短时间内再次提问，也就是关联上下文】
- **功能3**: 【只有一个悬浮窗口，包含一个输入框和随输出内容伸缩的输出框，输出框是可以渲染MarkDown内容】
- **功能4(扩展)**: 【保存用户输入历史和回复历史到本地History目录，格式为.md或者.mdx，并且支持关键字搜索历史】

# 技术与平台约束 (Constraints)
- **目标平台**: Windows 10/11 和 Linux
- **技术栈偏好**: 优先考虑使用 C++ 和 Qt6 框架进行开发，Gui可以使用qml】
- **团队规模**: 目前只有我一个人开发

# 任务
请为我生成一份详细、专业的软件开发规划。

# 输出格式
请使用 Markdown 格式进行输出，并必须包含以下章节：
1.  **项目概述 (Project Overview)**: 对项目的总结和目标。
2.  **核心功能模块拆解 (Feature Module Breakdown)**: 将上述功能需求细化为更小的开发模块。
3.  **技术选型建议 (Technology Stack Recommendation)**: 基于我的偏好，给出具体的技术选型方案和理由（例如：具体的库、数据库等）。
4.  **数据库设计 (Database Schema)**: 如果需要，设计核心数据表的结构。
5.  **里程碑与开发时间线 (Milestones & Timeline)**: 将开发过程划分为 3-4 个主要里程碑，并估算每个阶段的时间（可以按周为单位）。
6.  **潜在风险与应对策略 (Risk Analysis)**: 分析开发过程中可能遇到的技术或非技术风险，并提出建议。
